# Census Truth Project

A demographic database built on 100 years of US federal census records (1850-1950),
using IPUMS full-count data as the foundation for genealogical research and family
linkage across decades.

---

## The Problem

Most genealogy databases are built on names. Names are unreliable. They change,
get misspelled, get anglicized, get borrowed from geography, and get fabricated
by well-meaning family members. This project is built on demographic truth instead.

**DNA is the truth. Everything else is evidence.**

---

## What This Project Does

- Loads IPUMS full-count census data (1850-1950) into decade-specific SQLite databases
- Translates all numeric IPUMS codes to human-readable text via auto-generated getter modules
- Provides search and dump tools to query the data across all decades
- Will eventually generate GEDCOM files for upload to Ancestry.com, using their
  volunteer user base to attach names to anonymous demographic records
- Validates those name attachments against the original census ground truth

---

## Architecture

### Layer 1 - Ground Truth
Raw IPUMS census data loaded into SQLite. Primary key is `YEAR + SERIAL + PERNUM`.
No dependency on names. Approximately 50% of records have names; this project
treats names as optional decoration, not identity.

### Layer 2 - Code Translation
IPUMS stores everything as numeric codes. A code lookup system translates these
to human-readable text at query time:
- `IpumsCodebookParser.py` reads the IPUMS codebook (basic.txt) and auto-generates
  JSON lookup files and Python getter modules for every variable in the extract
- `build_county_json.py` handles county codes separately from the ICPSR county CSV
- Getter modules (`_get_stateicp.py`, `_get_city.py`, etc.) accept any numeric
  format and return descriptive text

### Layer 3 - Name Resolution (planned)
GEDCOM files generated from Layer 1 data will be uploaded to Ancestry.com.
Ancestry's volunteer community will attach names and family knowledge to the
anonymous demographic records. Those attachments will be extracted and validated
against Layer 1 ground truth, producing a confidence score for each name match.

---

## Data Source

**IPUMS USA** - https://usa.ipums.org
Full-count census databases 1850-1950. These are the complete enumeration records,
not samples. Each decade database contains every person recorded in that census.

The IPUMS codebook file (basic.txt) describes all variables in a given extract.
Running IpumsCodebookParser.py against that file generates all required lookup
infrastructure automatically.

## Setup and Usage

### Requirements
- Python 3.8+
- No external packages required - uses only Python standard library

### Database Location
SQLite databases live on a local drive and are not included in this repository
due to size (2GB to 20GB per decade). Download full-count data from IPUMS USA
and run the loading script to build the vaults.

### First Run
1. Download your IPUMS extract and place basic.txt in data/
2. Place icpsrcnt.csv in data/
3. Run IpumsCodebookParser.py to generate all JSON and getter files
4. Run build_county_json.py to generate county lookup files
5. Run DumpVaultData.py to query the vaults

### Searching
Edit the target_surname variable in DumpVaultData.py:

```python
target_surname = "ASKEY"    # specific surname
target_surname = "%"        # all records (use with caution on full databases)
```

Results are written to timestamped files in the Output/ directory.

## Primary Key

Every person in every census year is uniquely identified by:

```
YEAR + SERIAL + PERNUM
```

SERIAL identifies the household. PERNUM identifies the individual within
that household. Together with the census year, this is a stable, reliable
unique identifier that does not depend on names.

---

## Notes

- The 1890 census was largely destroyed by fire and is not available
- Names are present in approximately 50% of records and are treated as
  supplementary data, not primary keys
- County-level geography (COUNTYICP) uses ICPSR codes which require the
  separate county lookup file, not the main codebook
